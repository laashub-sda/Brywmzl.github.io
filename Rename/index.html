Rename - 批量重命名工具

Rename 是一款运用内置规则命令静默批量重命名的工具


Get it!


更新日志


更新计划
- 新增 命令 “conver”（用于大小写转换）
- 新增 命令 “get”（用于截文本长度）[起始位置;取长度]
- 新增 命令 “if”（用于判断，引导分支）
- 新增 文件操作功能（复制、移动、删除）
- 新增 预判功能（判断往后的命名结果是否冲突）
- 新增 文件名称校检（是否有特殊字符，是否符合长度）
- 添加 进度条（有终止和忽略选项）
- 支持对变量进行变更

20002（2020-01-02）
- 修复 导入列表时命令为空报错
- 修复 commands 目录不在无法保存预设

20001（2020-01-01）元旦快乐！
- 新增 命令 “set_value”
- 新增 常量 “get” “file_size” “image_width” “image_height”
- 新增 命令参数类型（text、match、get，参数类型）
- 新增 列表预览功能
- 新增 命令预设文件（.txt 左键保存右键删除）
- 去除 “clean.extension” (set_extension 可替代)
- 去除 右键预设（改为插入常量）
- 去除 拖拽面板（文件载入后，需要按 “start” 执行）
- 修复 大文件经过校检导致内存不足
- 修复 “ddd” 为（1~365）
- 修复 没有预设时开启列表数组小于1报错
- 修复 多个相同常量处理问题
- 优化 正则表达式辨识度（支持匹配子集）
- 本次更新的内容不少，文件也大了许多 ~~~

19277（2019-10-04）国庆假期过去了一半了！
- 本次更新的最大亮点：从单行的 Rule 变为多行的 Command
- 这次更新基本上重写，所以也就重新编译了，文件体积变小了很多
- 修改了 “index” 补零的方式（<index,3,3> 参数.1.补零数 2.起始索引）
- 修复 “ddd” 生成错误
- 修复 导入多个文件无法识别的问题
- 去掉了 “sequence” 感觉没啥用，就去掉了
- 新增 常量 “md5” “sha1” “sha256” “sha512”
- 命令 “set” “rep” “clean”
  set 分支: “set.match” “set.extension”
  rep 分支: “rep.match”
  clean 分支: “clean.match” “clean.extension”




19012（2019-01-12）
- 变更了规则
- index 支持补零
- 支持扩展名重写
- 支持将文件创建为副本
- 支持正则表达式替换
- 支持保存成配置

0.0 忘记什么时候更新的了~
- 修复 windows 10 不能拖拽问题
- 支持 “index” 变量
